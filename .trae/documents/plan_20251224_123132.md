## 实现计划

1. **设计错误码结构**
   - 使用枚举类实现错误码系统
   - 每个错误码包含三个属性：错误码（code）、错误信息（message）和 HTTP 状态码（httpStatus）
   - 提供构造函数支持自定义错误码

2. **分类定义错误码**
   - 系统级错误：如服务器内部错误、服务不可用等
   - 参数错误：如参数缺失、参数格式错误等
   - 业务逻辑错误：如资源不存在、操作失败等
   - 认证授权错误：如未登录、权限不足等
   - 数据库错误：如数据库连接失败、SQL 执行错误等

3. **添加常用错误码**
   - 20000：成功
   - 40000：请求参数错误
   - 40100：未授权
   - 40300：禁止访问
   - 40400：资源不存在
   - 50000：服务器内部错误
   - 50300：服务不可用
   - 等其他常用错误码

4. **提供自定义错误码支持**
   - 允许通过构造函数创建自定义错误码
   - 提供静态方法快速创建自定义错误码

5. **实现必要的方法**
   - getter 方法获取错误码属性
   - 重写 toString 方法方便日志输出
   - 提供静态方法根据错误码获取枚举值

6. **保持代码的可扩展性**
   - 使用枚举类便于后续添加新的错误码
   - 分类清晰，便于维护

## 预期结果

完成后，ErrorCode 枚举类将包含常用的 HTTP 接口错误码，支持自定义错误码，并提供方便的使用方式。其他组件可以直接使用这些错误码来统一处理 API 错误，提高代码的一致性和可维护性。
