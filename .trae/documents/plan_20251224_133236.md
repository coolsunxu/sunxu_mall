## 实现方案

MyBatis Generator 不直接支持为单个表配置不同的目标路径，但可以通过以下方式实现：

### 方案：使用多个 Context

为每组需要不同路径的表创建一个单独的 Context，每个 Context 可以有自己的生成器配置。

### 实现步骤

1. **拆分 Context**：
   - 为 `sys_user` 表创建一个 Context，配置其专属的包路径
   - 为 `sys_role` 表创建另一个 Context，配置其专属的包路径

2. **配置每个 Context**：
   - 为每个 Context 设置唯一的 `id`
   - 为每个 Context 配置独立的 `javaModelGenerator`（domain 包路径）
   - 为每个 Context 配置独立的 `sqlMapGenerator`（xml 路径）
   - 为每个 Context 配置独立的 `javaClientGenerator`（DAO 路径）
   - 每个 Context 只包含自己负责的表

3. **配置示例**：
   - 第一个 Context 处理 `sys_user` 表，生成到 `sys.web` 包
   - 第二个 Context 处理 `sys_role` 表，生成到 `sys.role` 包

### 预期结果

配置完成后，运行 MyBatis Generator 时：
- `sys_user` 表的实体类会生成到 `com.example.sunxu_mall.entity.sys.web`
- `sys_user` 表的 XML 映射文件会生成到 `com.example.sunxu_mall.mapper.sys.web`
- `sys_user` 表的 DAO 接口会生成到 `com.example.sunxu_mall.mapper.sys.web`
- `sys_role` 表的实体类会生成到 `com.example.sunxu_mall.entity.sys.role`
- `sys_role` 表的 XML 映射文件会生成到 `com.example.sunxu_mall.mapper.sys.role`
- `sys_role` 表的 DAO 接口会生成到 `com.example.sunxu_mall.mapper.sys.role`

### 注意事项

1. 每个 Context 必须有唯一的 `id`
2. 每个 Context 可以共享相同的数据库连接配置
3. 每个 Context 可以有自己的插件配置
4. 运行生成器时，所有 Context 会依次执行

### 实现代码

修改 `generatorConfig.xml`，添加多个 Context 配置，每个 Context 配置不同的生成路径和表。
